# サブエージェント委任の知識: 判断・指示文・検証

並列化そのものの規約（独立単位のみ・状態ファイルは主エージェント専任・TDD順序厳守など）は `.aidd-docs/rules/implementation.md` と `.aidd-docs/rules/workflow.md` にある。このファイルはその上で「委任するかどうかをどう判断し、どう指示文を書き、結果をどう扱えば品質が落ちないか」の実践知識とチェックリストである。

## 1. 委任するかどうかの判断

サブエージェントは会話の文脈を持たずゼロから開始する。**起動コスト＝文脈の再構築コスト**である。主エージェントがすでに把握している内容で数分で終わる作業は委任せず自分でやる。

委任が割に合うのは:
- (a) 大量のファイル読み込みが必要で「結論だけ」あればよい調査
- (b) 互いに独立した機能単位の実装
- (c) 観点を絞ったレビュー

「観点が複数ある」「徹底的に調べたい」はそれ自体では並列化の理由にならない。逐次で足りるなら逐次でよい。

**既定は主エージェント自身が実作業（調査・実装・執筆）を行うモードである。** 「指揮命令者かつレビュアー」に回って作業者へ委任するのは一つのモードにすぎず、基本は委任しない。委任へ切り替えるのは、(a) 上記のように委任が明らかに割に合うと判断したとき、または (b) 人間の指示があったとき に限る。委任モードに入ったら、主エージェント自身が手を動かすのは統合・裁定・小さな修正にとどめ、実作業は作業者に任せる（会話や仕様解釈の文脈を消費せずに済む）。

モデル階層の選び方（委任すると決めたあとの話）:
- **委任するなら、既定は自身より下位のモデルの作業者に委任する。** haiku でこなせる作業なら haiku でよい（下位モデルが少し難しい作業を担当してしまうのは許容範囲。こなせる見込みがあれば下げる）
- **難易度が高いと判断したら、自身と同等のモデルの作業者を使ってよい。**
- **自身より上位のモデルの能力が必要だと思ったら、勝手に進めず人間に打診する。** 主エージェントは自分でモデルを上げられないため、人間が「上位モデルへ切り替える」か「今のモデルで可能な限り尽くす」かを判断する

## 2. 指示文のチェックリスト（委任前に確認）

作業者は会話もプロジェクトルールも読んでいない前提で、指示文だけで作業が完結するかを確認する。

- [ ] 目的と完了条件（何ができたら終わりか）が1〜2文で書いてある
- [ ] 作業者が知り得ない文脈を書いた（会話で決まったこと、すでに試して駄目だったこと、採用しない選択肢とその理由）
- [ ] 変更してよいファイル範囲と、触ってはいけない範囲を明示した
- [ ] 検証方法（実行するコマンドと期待結果）を書いた
- [ ] プロジェクトの禁止事項を転記した（テストを弱めない／環境変更しない／ファイル書き換えはスクリプトでなく編集ツールで／履歴 specs を参照しない）
- [ ] 報告フォーマットを指定した（変更ファイル一覧・検証結果・未解決事項）。作業者から返るのは最終報告だけなので「途中でわかったことも最終報告に含める」ことを指示する
- [ ] 迷ったら勝手に決めず、選択肢と推奨を報告して作業を終えるよう指示した

## 3. 結果の受け取りと統合

- 作業者の「完了しました」を信用しない。主エージェントが検証コマンドを自分で実行してから統合する
- 作業者の報告は人間には見えない。人間に必要な内容は主エージェントが要約して伝える
- 複数の作業者の報告が矛盾したら、主エージェントが自分で該当コードを読んで裁定する。伝聞をそのまま人間や別の作業者に転送しない
- 作業者の待ち時間には主エージェントが次の準備（レビュー基準の整理・次の指示文の下書き）を進める

## 4. アンチパターン集

| アンチパターン | 症状 | 正しい形 |
|---|---|---|
| 丸投げ | 指示文に文脈がなく、作業者が推測で仕様を埋める | 決定事項・制約・却下済み案を指示文に書く |
| 無検証統合 | 作業者の自己申告だけで完了扱いにする | 主エージェントがテスト・ビルドを実行してから取り込む |
| 伝言ゲーム | 作業者Aの結論を検証せず作業者Bの前提として渡す | 主エージェントが裁定してから渡す |
| 同一ファイルの並列 | 複数の作業者が同じファイルを変更して上書き・衝突 | ファイル範囲を排他に割り当てる |
| 細切れ委任 | 小さいタスクを1つずつ別の作業者に委任し、毎回文脈の再構築コストを払う | まとまった機能単位で通しで任せる。継続できるエージェントには新規起動でなく継続を使う |
| 主エージェントの空回り | 委任したのに主エージェントが同じ作業を並行して自分でもやる | 委任したら任せ、主エージェントはレビュー基準の準備に回る |
